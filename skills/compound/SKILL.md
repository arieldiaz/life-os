# Compound

## Overview

Write back what was learned — about the work, the world, or the process itself — into `memory/`, and when the lesson is structural, into the skills and STRATEGY.md themselves. Skip everything else before you skip this: a loop without write-back is just activity.

## When to use

After /review hands over candidates; when a session surfaces a "huh, again?" moment; weekly, as ritual. Not for trivia — compounding noise is worse than not compounding.

## Process

1. **Take each candidate and ask: is this durable?** Durable = would change behavior next time a similar situation appears. "The API was flaky Tuesday" is weather. "I consistently underestimate integration work 3x" is climate. Only climate compounds.
2. **Merge before adding.** Search `memory/index.md` for a lesson this rhymes with. Strengthening an existing lesson with a second data point beats a near-duplicate — repetition across cycles is itself signal, so note the recurrence.
3. **Write the lesson file** (`memory/lessons/short-name.md`): the lesson in one bold sentence, the evidence (cite stream events/reviews — a lesson without provenance is a vibe), and what to do differently, concretely.
4. **Update `memory/index.md`** — one line per lesson. If the index is over ~2 pages, consolidate before adding.
5. **Escalate structural lessons.** If the lesson is about the *process* ("I over-plan when anxious", "reviews work better as voice notes"), propose an edit to the relevant skill file or STRATEGY.md's Operating notes. The framework is inside its own loop — this is the mechanism.
6. **Route patterns separately:** recurring shapes that aren't yet lessons ("energy craters after context-switching days") go to `memory/patterns/` as observations awaiting enough evidence to become lessons.

## Rationalizations

| Excuse | Rebuttal |
|--------|----------|
| "I'll remember this" | You demonstrably won't, and the next agent session definitely won't. That's the entire problem this solves. |
| "More lessons = more compounding" | Memory that can't be loaded isn't memory. Curation IS the compounding. |
| "Editing the skills feels presumptuous" | The skills are drafts, permanently, by design. Propose the edit; the human decides. |
| "This lesson is embarrassing" | Embarrassing lessons are the highest-yield ones. Write it kindly and write it down. |

## Exit criteria

Durable lessons written with provenance, index current, structural lessons escalated as proposed edits, and the total memory *more* loadable than before — not just bigger.
